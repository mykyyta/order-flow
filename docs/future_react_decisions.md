# Future: React Decisions and Progress

Останнє оновлення: 2026-02-07

## Рішення (зафіксовано)

1. **Єдине джерело правди для статусів**
- Коди статусів, їхні людиночитні назви (UA) і правила переходів належать доменній моделі.
- Фронтенд (Django templates / React) не хардкодить українські назви статусів.
- UI отримує статуси з backend-контракту (API або серверний контекст) у вигляді:
  - `status_code`
  - `status_label`
  - `allowed_transitions`

2. **Межа відповідальності**
- Backend: доменні правила, статуси, валідація, переходи, формування даних для UI.
- Frontend: тільки відображення і взаємодія, без дублювання доменної логіки.

3. **React-напрям**
- React є цільовим UI-шаром для нових/перенесених екранів.
- Новий функціонал UI має орієнтуватися на API-контракти, а не на логіку в шаблонах.

## Що вже зроблено

1. Проведено аудит поточного UI/flow як базу для переходу на React.
2. Зафіксовано нове архітектурне рішення про статуси:
   - українські назви статусів не живуть у фронтенді як локальний хардкод;
   - доменна модель є `source of truth`.

## Статус реалізації

- У цьому кроці змінено лише документацію.
- Кодова імплементація рішень буде виконуватися окремими змінами.
